# Функциональные требования  
## Приложение: «Корпоративный бонусный калькулятор»  
Версия: 1.0  
Тип приложения: Web / Мобильное (без ролей, один тип пользователя — сотрудник)

---

## 1. Назначение приложения

Приложение позволяет сотруднику ввести данные о выполнении ключевых показателей эффективности (KPI) и автоматически рассчитать размер месячной премии с учетом:
- количества выполненных условий;
- весовой значимости каждого условия;
- критичности отдельных показателей.

---

## 2. Исходные данные (ввод пользователя)

| Поле | Тип | Значения | По умолчанию |
|------|-----|----------|--------------|
| Базовый бонус | число (руб.) | целое, 0 – 1 000 000 | 10 000 |
| План продаж | boolean | да / нет | нет |
| Отсутствие прогулов | boolean | да / нет | нет |
| Прохождение обучения | boolean | да / нет | нет |
| Соблюдение сроков отчетов | boolean | да / нет | нет |

---

## 3. Весовые коэффициенты условий

| Условие | Вес | Критичность |
|---------|-----|--------------|
| План продаж | 0.40 | 🔴 критическое |
| Отсутствие прогулов | 0.30 | 🔴 критическое |
| Прохождение обучения | 0.20 | 🟢 обычное |
| Соблюдение сроков отчетов | 0.10 | 🟢 обычное |
| **Сумма весов** | **1.00** | — |

---

## 4. Формула расчета премии

### 4.1. Основная формула

```
Премия = Базовый бонус × (Сумма весов выполненных условий) × Ккрит
```

### 4.2. Коэффициент критичности (Ккрит)

| Сценарий | Ккрит |
|----------|-------|
| Выполнены оба критических условия (план продаж + отсутствие прогулов) | 1.2 |
| Выполнено ровно одно критическое условие | 1.0 |
| Не выполнено ни одного критического условия | 0.7 |

### 4.3. Ограничения

- Если **сумма весов выполненных условий = 0** → премия = 0 (расчет не выполняется, выводится сообщение).
- Результат округляется **в меньшую сторону** до целых рублей (`Math.floor`).
- Максимальная премия = `Базовый бонус × 1.0 × 1.2` = `Базовый бонус × 1.2`

| План продаж | Отсутствие прогулов | Обучение | Сроки отчетов | Сумма весов | Ккрит | Итоговая премия (при базе 10 000 ₽) |
|-------------|---------------------|----------|---------------|-------------|-------|--------------------------------------|
| ✅ | ✅ | ✅ | ✅ | 1.00 | 1.2 | **12 000 ₽** |

| ✅ | ✅ | ❌ | ❌ | 0.70 | 1.2 | **8 400 ₽** |
| ✅ | ❌ | ✅ | ✅ | 0.70 | 1.0 | **7 000 ₽** |
| ✅ | ❌ | ❌ | ❌ | 0.40 | 1.0 | **4 000 ₽** |
| ❌ | ✅ | ✅ | ✅ | 0.60 | 1.0 | **6 000 ₽** |
| ❌ | ❌ | ✅ | ✅ | 0.30 | 0.7 | **2 100 ₽** |
| ❌ | ❌ | ❌ | ❌ | 0.00 | — | **0 ₽** (сообщение) |

## 6. Отображение результатов

На экране пользователь должен видеть:

| Элемент | Формат | Пример |
|---------|--------|--------|
| Количество выполненных условий | текст | `Выполнено: 3 из 4` |
| Сумма весов выполненных условий | число (дробь) | `Сумма весов: 0.70` |
| Коэффициент критичности | число | `Ккрит: 1.2` |
| Итоговая премия | целое число + ₽ | `Премия: 8 400 ₽` |
| Детализация по каждому условию | список с весами и галочками | `✅ План продаж (вес 0.40, критическое)` |
| Особое сообщение | текст + иконка | при премии = базе × 1.2 → `🏆 Максимальная премия с бонусом за критичность` |

---

## 7. Валидации и ограничения

| Правило | Действие |
|---------|----------|
| Базовый бонус < 0 | установить 1000 |
| Базовый бонус = 0 | установить 1000 |
| Базовый бонус > 1 000 000 | ограничить ввод (max=1 000 000) |
| Все условия = false | показать `Премия не начисляется (выполнено 0 условий)` |
| Любое изменение полей | автоматический пересчет (без кнопки «Рассчитать») |

---

## 8. Интерфейс и управление

| Элемент | Требование |
|---------|-------------|
| Чекбоксы | для каждого условия с подписью и иконкой критичности (🔴 / 🟢) |
| Поле ввода | числовое поле для базового бонуса |
| Кнопка «Сбросить все» | сбрасывает все чекбоксы в false, базовый бонус → 10 000 |
| Адаптивность | корректное отображение от 320px до 1920px |
| Доступность | управление с клавиатуры (Tab, пробел, Enter) |

---

## 9. Нефункциональные требования

| Параметр | Значение |
|----------|----------|
| Время пересчета | < 50 мс |
| Платформа | Web (HTML/CSS/JS) или React / Vue / Flutter |
| Хранение данных | опционально — `localStorage` для восстановления последнего состояния после перезагрузки |
| Округление | всегда `Math.floor` (вниз) |

---

## 10. Блок-схема логики (текстовое описание)

```
Начало
  ↓
Получить базовый бонус
  ↓
Получить состояния 4 условий
  ↓
Рассчитать сумму весов выполненных условий
  ↓
Если сумма весов == 0 → вывести "Премия 0 ₽"
  ↓
Иначе:
    Определить Ккрит:
        - если план продаж ✅ и прогулы ✅ → 1.2
        - если одно из двух критических ✅ → 1.0
        - иначе → 0.7
    ↓
    Премия = базовый_бонус × сумма_весов × Ккрит
    ↓
    Округлить вниз (floor)
    ↓
    Вывести результат + детали
  ↓
Конец
```

## 11. Критерии приемки (готового приложения)

- [ ] Все поля ввода работают согласно таблице валидаций.
- [ ] Премия пересчитывается при любом изменении данных.
- [ ] Примеры из раздела 5 дают указанные результаты.
- [ ] Критические условия визуально отмечены 🔴.
- [ ] При всех критических условиях + всех обычных показывается сообщение о максимальной премии.
- [ ] Кнопка «Сбросить все» корректно сбрасывает форму.
- [ ] Приложение корректно работает на экране 375px (мобильный вид).
